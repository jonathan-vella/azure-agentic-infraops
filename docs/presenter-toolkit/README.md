## üé§ Demo Preparation Checklist

### Before the Demo

- [ ] Test GitHub Copilot subscription is active
- [ ] VS Code with GitHub Copilot extension installed and signed in
- [ ] Azure subscription with Contributor access (for deployment demos)
- [ ] Dev Container loads successfully (3-5 minutes first time)
- [ ] Practice the 30-minute demo script at least once
- [ ] Prepare answers for common objections

### Demo Day

- [ ] Close unnecessary applications (notifications, email, chat)
- [ ] Use a clean VS Code window (no distracting tabs)
- [ ] Have backup screenshots/recordings in case of connectivity issues
- [ ] Test microphone and screen sharing before starting

## üéØ Audience-Specific Tips

### Presenting to IT Professionals

- Focus on **practical time savings** with real Infrastructure-as-Code examples
- Show how Copilot **teaches best practices** (naming conventions, security defaults)
- Demonstrate the **iterative workflow** (describe intent ‚Üí accept suggestions ‚Üí refine)

### Presenting to Leadership/Executives

- Lead with **ROI and business impact** (use the ROI Calculator)
- Highlight **risk reduction** (fewer misconfigurations, built-in security)
- Emphasize **skill acceleration** for new team members

### Presenting to Developers (Expanding to IaC)

- Show how Copilot **bridges the gap** from application code to infrastructure
- Demonstrate **context awareness** (Copilot understands the full stack)
- Highlight **reduced context switching** (stay in VS Code for everything)
